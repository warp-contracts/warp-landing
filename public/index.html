Blockchain data problem

Most modern smart contracts treat data as a side-product that slows down processing. Data is digested, pruned and pushed out to side chains. Nonetheless the bloated global state is still the biggest challenge of modern blockchains.

How SmartWeave solves that

We use a radically different design treating data as the most valuable asset. We get rid of a global state thanks to decoupling the storage from the computation. The contracts can be evaluated on-demand, lazily, creating the potential for unparalleled scalability.

Architecture

To improve blockchain performance, smart contracts evaluation is taken out from the blockchain and handed over to client-side. That's where this SDK comes into play- wrapping up complicated contract interaction issues into few easy to use library methods. All written in modular fashion, allowing you to write and integrate your own modules, and ready to use with any cache mechanism.

Outstanding performance

Optimization of contract evaluation and caching mechanisms allowed us to reach speed of contract evaluation not yet seen on Arweave blockchain.
